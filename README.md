## 功能介绍

### 页面生成

- 监听酒馆正文回复完成事件，按设置自动生成页面。
- 支持手动生成、指定楼层生成、批量生成，以及对已有页面重新生成候选版本。
- 使用 `generateRaw` 直接请求模型，不读取当前酒馆预设提示词。
- 支持主 API 与第二 API；第二 API 支持 OpenAI 兼容、Google AI Studio、Google Vertex AI。
- 支持流式输出，并在面板里展示生成状态、请求日志和错误信息。
- 支持失败自动重试；默认关闭，默认超时 `360000ms`，最大重试次数 `3`。

### 提示词系统

- 内置基础提示词和小剧场提示词。
- 支持本地新增、编辑、删除、复制、发布与恢复内置小剧场。
- 支持随机小剧场：
  - 可选择多个候选小剧场。
  - 可设置每次抽取数量。
  - 可设置触发阈值；当前逻辑为 `roll >= probability` 时触发请求，否则跳过本次生成。
- 支持一次性小剧场：临时加入本次生成，不写入长期库。
- 支持提示词预览，能查看本次请求会发送的 system / assistant / user 内容。

### 来源采集

- 采集当前聊天可见楼层、最新用户消息、最新 AI 回复。
- 读取角色卡、用户设定、绑定世界书、激活世界书条目。
- 支持世界书条目纳入/排除开关，并显示世界书总览和本次纳入字数估算。
- 自动提取历史页面中的正文记忆，用于后续页面衔接。
- 对隐藏楼层、不可见内容和被排除内容做过滤，避免误纳入上下文。

### 页面管理

- 每个聊天保存独立页面记录。
- 每条页面记录支持多个候选版本，可前后切换。
- 支持收藏、删除、标题编辑、源码编辑和刷新预览。
- 支持从原始酒馆楼层跳回对应消息。
- 会在酒馆消息楼层旁添加打开按钮，用于快速打开该楼层对应页面。
- 页面主体优先存入 IndexedDB，外壳索引保存在 localStorage / 聊天 metadata 中，以降低 localStorage 配额压力。

### 面板与入口

- 悬浮球入口：可拖拽、可贴边、可自定义图标/颜色/大小。
- QR 按钮入口和扩展菜单入口：可在设置中开关。
- 面板支持拖拽、缩放、全屏、移动端强制全屏。
- 支持日间/夜间/定时主题，并可自定义面板与悬浮球配色。

### 运行设置

- 自动生成开关。
- 聊天历史读取深度。
- 手机端页面缩放。
- 本地缓存上限和当前聊天/全部缓存清理。
- 排除角色卡名单。
- 排除标签，内置标签包含 `echo`、`ta的手机`、`gossip`、`danmu`、`幕后故事`、`日月来信`、`branches`、`horae`。
- 第二 API 多配置档管理、模型列表读取、API 密钥与代理密码保存。
- 脚本初始化与格式化脚本/当前聊天窗口。

## 运行流程

1. `index.ts` 用 `?raw` 读取本目录下的 TypeScript 片段。
2. 所有片段按固定顺序拼接进一个 IIFE。
3. 通过 `new Function()` 执行拼装后的脚本。
4. `host-buttons.ts` 启动挂载流程。
5. `ui-events.ts` 创建 iframe 面板、悬浮球和宿主入口。
6. `monitor.ts` 监听酒馆事件，判断正文生成是否完成。
7. `generation.ts` 调用 `getGenerationPrompt()`、采集来源、构建提示词并发送 `generateRaw` 请求。
8. `prompt-builder.ts` 解析模型返回的 HTML。
9. `online-store.ts` 保存页面、候选版本、正文记忆和缓存索引。
10. `ui-render.ts` 重新渲染页面预览、设置面板、来源面板和调试信息。

```text
├─ source-collector.ts       # 聊天、角色卡、世界书、历史页面正文记忆采集
├─ prompt-builder.ts         # 请求提示词构建、HTML 解析、正文记忆提取、提示词预览
├─ generation.ts             # 第二 API、模型列表、generateRaw 请求、重试和生成主流程
├─ monitor.ts                # 酒馆事件监听、正文完成检测、自动触发生成
├─ iframe-styles.ts          # 宿主悬浮球样式、面板尺寸、主题变量、拖拽/缩放工具
├─ iframe-template.ts        # iframe 内部完整 HTML/CSS 模板
├─ ui-events.ts              # UI 事件绑定、窗口挂载、交互逻辑、销毁清理
└─ host-buttons.ts           # 页面加载/卸载入口，调用 mount 与清理逻辑
```

## 关键数据

### 设置数据

设置主要由 `settings.ts` 管理，保存到脚本变量与本地存储中。主要包含：

- 当前基础提示词、小剧场、摘要标签选择。
- 自动生成、历史深度、移动端缩放。
- 外观主题与悬浮球样式。
- 第二 API 配置与配置档列表。
- 随机小剧场设置。
- 排除角色与排除标签。
- 本地缓存上限和打开入口设置。

### 页面数据

页面数据主要由 `online-store.ts` 管理，包含：

- 当前聊天的页面记录。
- 每条页面的候选版本、收藏状态、标题、来源楼层。
- 页面 HTML 与提取出的正文记忆。
- 聊天维度索引、角色维度索引、旧版本数据迁移。
- IndexedDB payload 与 localStorage 外壳索引。

### 调试数据

运行时日志保存在内存中，并显示在「调试」页：

- 监听事件。
- 来源采集结果。
- API 请求摘要。
- 解析、保存、重试和错误信息。
